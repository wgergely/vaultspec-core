# rag plan: GPU-Accelerated Local RAG for Vault Semantic Search

## Goal

Implement a local, GPU-accelerated (CPU-fallback) semantic search system for the `.docs/` vault. The system indexes all vault documents into a persistent vector store, enabling LLMs and users to query project development history by meaning rather than keywords alone. Builds on the existing vault API established in [[2026-02-08-vault-api-plan]].

## Architecture Overview

```
docs.py CLI (existing)
  |
  +-- audit     (existing, unchanged)
  +-- create    (existing, unchanged)
  +-- index     (new)  -->  VaultIndexer --> EmbeddingModel --> VaultStore (LanceDB)
  +-- search    (new)  -->  QueryParser --> HybridSearcher --> GraphReranker --> results
```

New modules under `.rules/lib/src/vault/`:

```
vault/
  __init__.py        (existing)
  models.py          (existing, extend with RAG types)
  scanner.py         (existing, reuse as-is)
  parser.py          (existing, reuse as-is)
  links.py           (existing, reuse as-is)
  hydration.py       (existing, unchanged)
  embeddings.py      (NEW - embedding model wrapper)
  store.py           (NEW - LanceDB connection and schema)
  indexer.py          (NEW - indexing pipeline)
  search.py          (NEW - retrieval pipeline)
```

## Phase 2: Embedding Model Wrapper (`vault/embeddings.py`)

Ref: [[2026-02-12-rag-embeddings-adr]]

- [ ] Create `.rules/lib/src/vault/embeddings.py`
- [ ] Implement `EmbeddingModel` class:

  ```python
  class EmbeddingModel:
      MODEL_NAME = "nomic-ai/nomic-embed-text-v1.5"
      DIMENSION = 768
      DOCUMENT_PREFIX = "search_document: "
      QUERY_PREFIX = "search_query: "
  ```

- [ ] `__init__(self, device: str | None = None)`:
  - Auto-detect GPU: `device = "cuda" if torch.cuda.is_available() else "cpu"`
  - Allow explicit override via parameter.
  - Load model via `SentenceTransformer(MODEL_NAME, device=device, trust_remote_code=True)`.
  - Log device selection: `"Embedding model loaded on {device}"`.
- [ ] `encode_documents(self, texts: list[str]) -> np.ndarray`:
  - Prepend `DOCUMENT_PREFIX` to each text.
  - Batch encode via `self.model.encode(prefixed, show_progress_bar=True, normalize_embeddings=True)`.
  - Return numpy array of shape `(n, 768)`.
- [ ] `encode_query(self, query: str) -> np.ndarray`:
  - Prepend `QUERY_PREFIX`.
  - Encode single text, return shape `(768,)`.
- [ ] `@property device(self) -> str`: Return current device string.

### 2.2 Device detection utility

- [ ] Implement `get_device_info() -> dict` utility function:
  - Returns `{"device": "cuda"|"cpu", "gpu_name": str|None, "vram_mb": int|None}`.

## Phase 3: Vector Store Layer (`vault/store.py`)

Ref: [[2026-02-12-rag-vectordb-adr]]

### 3.1 LanceDB schema definition

- [ ] Create `.rules/lib/src/vault/store.py`
- [ ] Define Pydantic schema for the LanceDB table:

  ```python
  class VaultDocument(LanceModel):
      id: str               # document stem (e.g., "2026-02-12-rag-plan")
      path: str             # relative path (e.g., "plan/2026-02-12-rag-plan.md")
      doc_type: str         # "adr", "plan", "exec", "research", "reference"
      feature: str          # feature tag without # (e.g., "rag")
      date: str             # ISO date from frontmatter
      tags: str             # JSON-serialized list of tags
      related: str          # JSON-serialized list of related wiki-links
      title: str            # H1 heading extracted from body
      content: str          # full markdown body (for BM25 full-text search)
      vector: Vector(768)   # embedding vector
  ```

  Note: `tags` and `related` stored as JSON strings because LanceDB list column support varies. Deserialized at query time.

### 3.2 `VaultStore` class

- [ ] `__init__(self, vault_root: pathlib.Path)`:
  - `self.db_path = vault_root / ".lance"`
  - `self.db = lancedb.connect(str(self.db_path))`
  - `self.table_name = "vault_docs"`
- [ ] `ensure_table(self) -> lancedb.Table`:
  - Create table if not exists with `VaultDocument` schema.
  - Create full-text search index on `content` column (Tantivy) if not exists.
  - Return table reference.
- [ ] `upsert_documents(self, docs: list[VaultDocument])`:
  - Delete existing rows by `id` match, then add new rows.
  - This handles incremental updates (modified documents).
- [ ] `delete_documents(self, ids: list[str])`:
  - Remove rows by `id` for documents that have been deleted from the vault.
- [ ] `get_all_ids(self) -> set[str]`:
  - Return set of all document `id` values currently in the store.
- [ ] `count(self) -> int`:
  - Return total number of indexed documents.
- [ ] `hybrid_search(self, query_vector: np.ndarray, query_text: str, filters: dict | None, limit: int) -> list[dict]`:
  - Execute LanceDB hybrid search combining:
    - ANN vector search on `vector` column.
    - BM25 full-text search on `content` column.
  - Apply metadata pre-filters as LanceDB `where` predicates:
    - `doc_type == X`, `feature == X`, `date LIKE 'prefix%'`.
  - Use `rerank` with `RRFReranker()` for reciprocal rank fusion.
  - Return list of result dicts with all columns plus `_relevance_score`.

## Phase 4: Indexing Pipeline (`vault/indexer.py`)

### 4.1 Document preparation

- [ ] Create `.rules/lib/src/vault/indexer.py`
- [ ] Implement `prepare_document(path, root_dir) -> VaultDocument | None`:
  - Read file content via `path.read_text(encoding="utf-8")`.
  - Parse metadata via existing `parse_vault_metadata(content)` from `vault/parser.py`.
  - Extract `doc_type` via existing `get_doc_type(path, root_dir)` from `vault/scanner.py`.
  - Extract feature tag: first non-directory tag from `metadata.tags`.
  - Extract title: first `#` heading from body, or filename stem as fallback.
  - Construct document text for embedding: `f"{title}\n\n{body}"`.
  - Return `VaultDocument` (without `vector` field -- filled during embedding step).

### 4.2 `VaultIndexer` class

- [ ] `__init__(self, vault_root: pathlib.Path, model: EmbeddingModel, store: VaultStore)`.
- [ ] `full_index(self) -> IndexResult`:
  - Scan all vault documents via `scan_vault(vault_root)` from `vault/scanner.py`.
  - Prepare all documents via `prepare_document()`.
  - Batch-embed all document texts via `model.encode_documents()`.
  - Assign vectors to documents.
  - Clear existing table and write all documents via `store.upsert_documents()`.
  - Return `IndexResult(total=N, added=N, updated=0, removed=0, duration_ms=T)`.
- [ ] `incremental_index(self) -> IndexResult`:
  - Scan all vault documents and compute file mtimes.
  - Compare against stored document ids via `store.get_all_ids()`.
  - Identify: new files (in vault, not in store), modified files (mtime newer than last index), deleted files (in store, not in vault).
  - Store last-indexed mtimes in a sidecar file: `{vault_root}/.lance/index_meta.json`.
  - Prepare and embed only new/modified documents.
  - Upsert changed documents, delete removed documents.
  - Return `IndexResult` with counts.
- [ ] `IndexResult` dataclass:

  ```python
  @dataclass
  class IndexResult:
      total: int          # total docs in index after operation
      added: int          # newly indexed
      updated: int        # re-indexed (modified)
      removed: int        # removed from index
      duration_ms: int    # wall-clock time
      device: str         # "cuda" or "cpu"
  ```

## Phase 5: Retrieval Pipeline (`vault/search.py`)

Ref: [[2026-02-12-rag-retrieval-adr]]

### 5.1 Query parsing

- [ ] Create `.rules/lib/src/vault/search.py`
- [ ] Implement `parse_query(raw_query: str) -> ParsedQuery`:

  ```python
  @dataclass
  class ParsedQuery:
      text: str                    # natural language portion
      filters: dict[str, str]      # extracted metadata filters
  ```

  - Extract filter tokens: `type:adr`, `feature:rag`, `date:2026-02`, `tag:#research`.
  - Remaining tokens form the natural language `text`.
  - Example: `"type:adr feature:rag vector database"` -> `ParsedQuery(text="vector database", filters={"doc_type": "adr", "feature": "rag"})`.

### 5.2 Graph-aware re-ranking

- [ ] Implement `rerank_with_graph(results: list[SearchResult], graph: VaultGraph, query: ParsedQuery) -> list[SearchResult]`:
  - Load `VaultGraph(vault_root)` from existing `graph/api.py`.
  - **Authority boost**: For each result, look up its in-link count from `graph.nodes[doc_id].in_links`. Apply multiplier: `score *= (1 + 0.1 * min(in_link_count, 10))`.
  - **Neighborhood boost**: If `query.filters` contains a `feature`, boost results whose wiki-link neighbors share that feature tag. Apply: `score *= 1.15` for neighborhood overlap.
  - **Recency boost** (mild): `score *= (1 + 0.02 * recency_rank)` where `recency_rank` is position when sorted by date descending (most recent = highest rank).
  - Re-sort by adjusted score.

### 5.3 `SearchResult` and `VaultSearcher` class

- [ ] `SearchResult` dataclass:

  ```python
  @dataclass
  class SearchResult:
      id: str
      path: str
      title: str
      doc_type: str
      feature: str
      date: str
      score: float
      snippet: str        # first 200 chars of content
  ```

- [ ] `VaultSearcher.__init__(self, vault_root, model: EmbeddingModel, store: VaultStore)`.
- [ ] `search(self, raw_query: str, top_k: int = 5) -> list[SearchResult]`:
  - Parse query via `parse_query()`.
  - Encode query text via `model.encode_query()`.
  - Execute `store.hybrid_search()` with vector, text, and filters.
  - Build `SearchResult` list from raw results.
  - Apply graph re-ranking via `rerank_with_graph()`.
  - Return top-K results sorted by final score.
